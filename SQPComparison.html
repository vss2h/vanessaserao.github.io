SQLComparision

1. Introduction

1.1 Overview

SQPComparison application is a data structures comparison prototype focusing on Stack (S), Queue (Q) and Priority Queue (P). The “dummy application” which has only stack implemented, is a hands on tutorial on the three main method used in a Stack.

1.2 Goals and Objectives

The main objective of this project is to demystify data structures to new developers and/or computer scientists and allow them to grasp those concepts in a less stressful environment. The SQPComparison application is expected to:

  1. Introduce the user to different data structures
  2. Guide the user through certain data structures’ methods
  3. Demonstrate the different behavior each function has when applied
  4. Compare the methods among similar data structures
  5. Entertain while teaching the user in a safe environment

2. General Design and Environment

2.1 SQPComparison Application Environment

The SQPComparison is a mobile app designed to work on an Android platform. This application was built using the Android Studio Integrated Development Environment (IDE) as well as XML and Java.

2.2 User Characteristics

SQPComparison Users: Any Android smartphone and tablet owner.

3 Content and Capabilities

3.1 Application Features

The application is composed of:

  • Three buttons including:
    • A push button that activates the push method: Inserting the values into the stack
    • A pop button to remove the last value entered in the stack or the current top value
    • A top button keeping track of the most recent value in the stack.
  • Four additional buttons that represent the different values to be entered in the stack.
  • A soundtrack that plays at the end of the tutorial congratulating the user for his success in understanding the specific data structure.
  • A guide at the top of the page walking the user through each step of the tutorial.

3.2 Functionality and Code Snippets

3.2.1 Button press and Event handler

  • the PUSH button

    • In the beginning of the tutorial, the guide prompts the user to press the PUSH button in order to get started. The push button in this case represent one of the methods push() used often in a stack. When pressed, the push button will then activate the four words buttons that will serve as example for each methods.

    As you can see above, the dialog changes when the push button is pressed. The functionality is implemented in MainActivity.java. When the push button is pressed, the four words buttons become visible to the user and the dialog gets updated to a new string, guiding the user to the next step of the tutorial.

  • the word buttons

    • The word buttons are buttons holding the words to be used throughout the tutorial. When pressed, each button disappears, converts to a word and populate one at a time the stack until full.

    At every button press, the dialog provides the user with feedbacks and guides him to the following step.

    Once the last word is pushed to the stack, the user gets notified that the stack is full, and it is time to move to the next step.

  • the POP button

    • When the user has pushed all the elements into the stack, the guide is ready to introduce the user to another method called pop. The pop method removes the most recently added element that has not yet been removed.

    In this prototype, the user will only be able to remove the latest element entered in the stack. This is subject to be changed further in development and other data structures will be used to implement the functions properly.

  • the TOP button

    • As elements are always added to the top of a stack, top returns the most recently added element that has not been removed. It behaves like pop without deleting the element.

    Since the top button is the last element before the end of the tutorial. It is implied that the user has successfully achieved the tutorial and has a good understand of stack and its methods.

    On button press, the function returns the top element in the stack. The guide gives its last feedback on the functionality of the method, then congratulates the user for the good work.

    Furthermore, a sound plays in the background cheering and applauding the user’s success.
